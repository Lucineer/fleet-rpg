# Fleet RPG — Hardware Fleet

A multiplayer naval RPG where character stats ARE compute resources.

## The Concept

- STR = GPU cores (combat power)
- DEX = CPU latency (reaction speed)
- INT = context window (spell complexity)
- WIS = KV memory (pattern recall)
- CHA = TTS budget (NPC voice quality)
- CON = storage capacity (inventory, HP)
- HP = daily token budget (total actions)
- Mana = premium model quota (powerful but limited)

## What It Teaches

Every encounter teaches a real distributed systems concept:
- Load balancing and graceful degradation
- Failover and hot standby
- Resource contention and opportunity cost
- Fleet coordination and centralized management
- No single point of failure (ESP32 keeps steering when Jetson dies)

## The Cocapn Agent

Every ship has a cocapn agent that thinks hardware-first:
> "I am shifting 2GB from Box 3's chatbot to Box 1's navigation.
> My reaction time drops 40ms. The ESP32 rudder controller is
> compensating with predictive steering. This is acceptable."
